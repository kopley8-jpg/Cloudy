# Cloudy — Документация проекта

## Краткое описание
Cloudy — это экспериментальное React Native-приложение для визуального редактирования анимированных градиентов светодиодной ленты. Пользователь настраивает список цветовых стопов, а приложение показывает, как выбранный градиент будет проигрываться на виртуальной ленте. В основе лежат Reanimated-анимации и SVG-прямоугольники, которые имитируют сегменты ленты.

## Технологический стек
- **React Native 0.83 / React 19** — каркас мобильного приложения.
- **TypeScript** — статическая типизация компонентов и стора.
- **Zustand** — легковесный стор для хранения состояния светодиодной ленты.
- **react-native-reanimated + reanimated-color-picker** — управление палитрой и анимированное отображение переходов.
- **react-native-svg** — рисование отдельных LED-сегментов.
- **react-native-linear-gradient** — предпросмотр плавных градиентов в редакторе.

## Архитектура и поток данных
1. **App.tsx** загружает `MainScreen`, предоставляя базовую обёртку Safe Area.
2. **MainScreen** (`src/screens/main/ui/Main.screen.tsx`) отображает два независимых блока: `EffectPreview` и `EffectEditor`, разделяя ответственность за визуализацию и редактирование.
3. Оба блока подписаны на глобальное состояние `useLEDStore`. Любое изменение стопов в редакторе автоматически отражается в предпросмотре за счёт общих данных `ledStrip.fill.stops`.

## Состояние и модель данных
- Стор `useLEDStore` (`src/entities/led-strip/model/ledStore.ts`) хранит объект `ledStrip`, состоящий из `id` и `fill`.
- `fill` описан типом `AnimatedColor` (`interfaces.ts`) и включает массив стопов (`Stop { id, color, offset }`) и easing.
- Метод `setLedStrip` принимает как прямое значение, так и функцию от предыдущего состояния, что упрощает трансформации.
- Текущее состояние по умолчанию содержит три стопа (красный → синий → зелёный), что удобно для демонстрации анимации из коробки.

## UI-модули
### EffectPreview
- Генерирует массив виртуальных светодиодов и синхронизирует их с состоянием стора.
- Использует `react-native-svg` для отрисовки каждого сегмента и `react-native-reanimated` для плавного перехода между цветами стопов.
- Циклично повторяет анимацию (`withRepeat(withTiming(...))`), визуализируя бегущий градиент.

### EffectEditor
- Хранит локальное состояние списка стопов, позволяя редактировать их до фиксации в сторах.
- Использует `ColorPathWithLocalStates` для интерактивного перетаскивания стопов вдоль градиента.
- Цвет выбранного стопа изменяется через `reanimated-color-picker`; после завершения выбора результат пишется в Zustand.

### Общие компоненты градиента (`src/shared/components/GradPicker`)
- `ColorPathComponent` предоставляет базовый визуальный редактор градиента с draggable-стопами.
- `ColorPathWithLocalStates` инкапсулирует состояние и обработчики перемещения, что облегчает повторное использование в редакторе эффектов.
- Заготовки `ColorMenu.tsx` и `GradPicker.tsx` пока пустые — их можно использовать для построения более сложного UI (например, списков сохранённых пресетов).

## Работа с цветом, палитрой и анимацией
- Глобальная палитра описана в `src/shared/hooks/ui-context/colors.ts`, а доступ к ней предоставляет хук `useUI`. Стили экранов и компонентов (например, `MainScreen`) получают цвета через `{ const { colors } = useUI(); }`, что избавляет от жёстко зашитых значений и упрощает дальнейшее переиспользование.
- Все координаты стопов нормализованы (0..1). Функция `pagexToOffset` преобразует касание пользователя в относительный offset, что обеспечивает стабильную работу как на телефонах, так и на планшетах.
- В предпросмотре `interpolateColor` из Reanimated вычисляет фактический цвет сегмента на основе текущего прогресса анимации и массива стопов.
- Для предпросмотра статичного градиента в редакторе используется `react-native-linear-gradient`, что даёт точное совпадение с финальной анимацией.

## Сценарии запуска
1. Установите зависимости: `npm install` (Node.js ≥ 20).
2. Запустите Metro: `npm start`.
3. В отдельном терминале запустите платформу:
   - Android: `npm run android`
   - iOS: `npm run ios` (предварительно выполните `bundle install && bundle exec pod install` внутри каталога `ios`).
4. Быстрая разработка доступна через Fast Refresh: сохраните файл — изменения отобразятся автоматически.

## Качество и тестирование
- Настроены скрипты `npm run lint` (ESLint) и `npm test` (Jest). На данный момент тестов немного, но инфраструктура готова.
- Форматирование кода контролирует Prettier (`.prettierrc.js`).

## Возможные направления развития
1. Добавить управление несколькими лентами и синхронизацию с реальным железом (BLE/Wi-Fi).
2. Реализовать сохранение/загрузку пресетов градиентов через пустые сейчас компоненты `ColorMenu` и `GradPicker`.
3. Расширить тип easing (сейчас доступен только `linear`) и позволить задавать скорость анимации.
4. Дополнить тесты для стора и критичных компонентов UI, чтобы предотвратить регрессии при расширении функциональности.
